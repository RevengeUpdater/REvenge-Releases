Авто-удаление файла - короткое объяснение

ЧТО ПРОИСХОДИТ

1. Пользователь скачивает REvenge_Setup_1.5.0.exe
2. После скачивания он автоматически открывается
3. Запускается отдельный поток который:
   - Ждёт пока пользователь закроет окно установщика
   - Проверяет живёт ли процесс REvenge_Setup_1.5.0.exe
   - Когда процесс закрывается - удаляет файл
4. Пользователь ничего не делает - всё автоматически

КОД

std::thread([hw,filePath=g_savePath](){
    WatchAndDeleteFile(filePath);
    PostMessage(hw, WM_CLOSE, 0, 0);
}).detach();

Запускает независимый поток который:
- Следит за процессом REvenge_Setup_1.5.0.exe
- Ждёт его закрытия через WaitForSingleObject()
- Удаляет файл
- Закрывает наше приложение

ТЕХНИЧЕСКИ

- Нет cmd.exe окна (чисто Win32 API)
- Нет таймеров (ждёт реального закрытия процесса)
- Работает с кириллицей и пробелами в пути
- Отдельный поток (не блокирует UI)
- Если что-то пошло не так - ждёт 60 сек и удаляет

РЕЗУЛЬТАТ

Файл REvenge_Setup_1.5.0.exe удаляется из Downloads папки
после того как пользователь закрыл установщик.
