Auto-delete File - Short Explanation

WHAT HAPPENS

1. User downloads REvenge_Setup_1.5.0.exe
2. After download it launches automatically
3. Separate thread starts which:
   - Waits while user closes installer window
   - Checks if REvenge_Setup_1.5.0.exe process is alive
   - When process closes - deletes the file
4. User does nothing - everything automatic

CODE

std::thread([hw,filePath=g_savePath](){
    WatchAndDeleteFile(filePath);
    PostMessage(hw, WM_CLOSE, 0, 0);
}).detach();

Launches independent thread which:
- Monitors REvenge_Setup_1.5.0.exe process
- Waits for its closure via WaitForSingleObject()
- Deletes the file
- Closes our application

TECHNICALLY

- No cmd.exe window (pure Win32 API)
- No timers (waits for real process exit)
- Works with Cyrillic and spaces in path
- Separate thread (doesn't block UI)
- If something fails - waits 60 sec and deletes anyway

RESULT

File REvenge_Setup_1.5.0.exe deleted from Downloads
after user closes the installer.
